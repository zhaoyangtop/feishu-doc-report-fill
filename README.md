# 飞书文档运营报告填充工作流 Skill

> 支持 **月度 / 季度 / 半年度 / 年度** 运营报告的飞书文档表格自动填充

## 功能概览

- 📊 **多周期报告框架**：月报、季报、H1/H2/年度报告，各有差异化模板
- 🔗 **数据源自动识别**：从飞书文档中的电子表格引用和 Excel 附件自动提取数据
- 📈 **数据聚合分析**：Python 脚本按月/品类/渠道/店铺多维度聚合，计算 ROI、占比、环比、同比
- ✏️ **Block 级文档更新**：通过 lark-cli `docs +update` 精准替换飞书文档表格内容
- 📝 **内容撰写指南**：运营 Review / 小结 / Plan 各板块的标准化撰写要点

## 核心规则

1. **数据来源唯一性**：所有数据必须取自文档内附件，不得使用外部数据源
2. **参考已有内容**：参考文档中已写好的历史内容，沿用其逻辑和结构
3. **格式不变**：大标题、表格结构、附件引用不变，仅填充运营内容
4. **数据真实准确**：所有数字从附件原始数据中提取和计算

## 报告周期

| 周期 | 时间节点 | 数据范围 | Plan 时间跨度 |
|---|---|---|---|
| 月报 | 每月第1-2个工作日 | 上月 | 下月 |
| 季报 | 季后第3-5个工作日 | 3个月 | 下季度 |
| H1 报告 | 7月第1-2周 | 1-6月 | H2 |
| H2 报告 | 次年1月第1-2周 | 7-12月 | 次年 |
| 年度报告 | 次年1月第2-3周 | 全年12月 | 次年 |

## 技术栈

- **lark-cli**：飞书文档/电子表格操作
- **openpyxl**：Excel 附件解析
- **Python**：数据聚合与分析
- **飞书 XML**：文档内容格式

## 适用场景

当用户说：
- "更新飞书文档 H1 汇报"
- "填写月度运营报告"
- "更新季度报告表格"
- "填充飞书文档中的运营数据"

时自动触发。
