# 飞书文档运营报告填充工作流

## 适用场景
当用户要求更新飞书文档中的运营汇报表格（月报、季报、H1/H2/年度汇报），需要从文档内引用的附件（电子表格、Excel文件）提取数据并填充到文档表格的各个单元格中时触发。

## 核心规则（不可违反）
1. **数据来源唯一性**：所有数据必须取自文档内附件（电子表格 cite 标签、Excel 文件附件 cite 标签），不得使用外部数据源或编造数据
2. **参考已写内容**：参考文档中已写好的历史内容（如上月月报、上季度季报），沿用其逻辑和结构，优化填充到目标表格
3. **格式不变**：大标题不变、表格结构不变、附件引用不变，仅填充/替换运营内容单元格中的文本
4. **数据真实准确**：所有数字必须从附件原始数据中提取和计算，不得估算或编造

---

## 报告周期框架

### 一、月度报告（月报）

**时间节点**：每月第1-2个工作日完成上月月报填写。

**数据源**：
- 当月店铺销售数据汇总-财务口径（飞书电子表格）
- 当月店铺销售及流量汇总-店铺口径（飞书电子表格）
- 当月千川消耗汇总（飞书电子表格）
- 当月抖音视频号明细（Excel附件，含抖音/视频号分店铺sheet）

**表格结构**：

| Category | 运营内容 | 备注 |
|---|---|---|
| 月度销售进展 | 当月收入/利润/费率/达成率/环比/同比 + 分店铺收入利润占比 | |
| 运营Review | 【销售收入与运营费用】+【店铺运营】+【千川推广】+【其他】 | |
| 运营小结 | 好的方面（3-4点）+ 存在的问题（3-4点） | |
| AI专项进展 | AI应用列表（标题优化/主图调整/数据分析/内容生产） | |
| 下月运营Plan | 销售预估 + 费用预估 + 销售计划(1.2.3.4) + 运营动作(1.2.3.4) | |

**关键指标**：
- 收入达成率 = 实际收入 / 计划收入 × 100%
- 费率 = 运营费用合计 / 销售收入 × 100%
- 环比 = (本月 - 上月) / 上月 × 100%
- 同比 = (本月 - 去年同月) / 去年同月 × 100%
- 千川ROI = 千川成交金额 / 千川消耗金额

### 二、季度报告（季报）

**时间节点**：每季度结束后第3-5个工作日完成（Q1=4月、Q2=7月、Q3=10月、Q4=次年1月）。

**数据源**：
- 季度内各月店铺销售数据汇总（飞书电子表格，3个月数据）
- 季度内各月店铺销售及流量汇总（飞书电子表格，3个月数据）
- 季度内千川消耗汇总（飞书电子表格，3个月数据）
- 季度抖音视频号明细（Excel附件，Q1/Q2/Q3/Q4各一份）

**表格结构**：

| Category | 运营内容 | 备注 |
|---|---|---|
| 季度销售进展 | 季度收入/利润/费率/达成率/环比/同比 + 分店铺季度收入利润占比 + 季度内月度趋势 | |
| 运营Review | 【销售收入与运营费用】季度汇总分析 + 【店铺运营】季度渠道结构变化 + 【千川推广】季度投放效率 + 【其他】SOP沉淀 | |
| 运营小结 | 好的方面（4-5点）+ 存在的问题（4-5点） | |
| AI专项进展 | AI应用季度总结（标题优化/主图调整/数据分析/内容生产/投放优化） | |
| 下季度运营Plan | 季度销售预估 + 费用预估 + 销售计划(1.2.3.4) + 运营动作(1.2.3.4) | |

**关键差异（vs 月报）**：
- 增加季度内月度趋势分析（3个月环比趋势线）
- 增加Q1 vs Q2 / Q3 vs Q4 对比分析
- 品类分析按季度汇总消耗与ROI，识别效率变化
- 达人渠道分析按季度汇总，评估合作策略调整效果
- 运营Plan时间跨度为3个月，需分月列示重点节奏

### 三、半年度/年度报告（H1/H2/年度）

**时间节点**：
- H1报告：7月第1-2周完成（覆盖1-6月）
- H2报告：次年1月第1-2周完成（覆盖7-12月）
- 年度报告：次年1月第2-3周完成（覆盖全年12个月）

**数据源**：
- H1/H2/全年店铺销售数据汇总（飞书电子表格）
- H1/H2/全年店铺销售及流量汇总（飞书电子表格）
- H1/H2/全年千川消耗汇总（飞书电子表格）
- Q1+Q2 / Q3+Q4 抖音视频号明细（Excel附件，两份合并）

**表格结构**：

| Category | 运营内容 | 备注 |
|---|---|---|
| 年度销售进展 | H1/H2/全年收入/利润/费率/达成率/同比 + 分店铺收入利润占比 + H1 vs H2对比 + 年度达成进度 | |
| 运营Review | 【销售收入与运营费用】半年/年度汇总 + 【店铺运营】渠道结构演变 + 【千川推广】投放效率趋势 + 【其他】SOP与方法论沉淀 | |
| 运营小结 | 好的方面（5-6点）+ 存在的问题（5-6点） | |
| AI专项进展 | AI应用半年/年度总结（标题优化/主图调整/数据分析/内容生产/投放优化/自动化流程） | |
| H2/下年度运营Plan | 半年/年度销售预估 + 费用预估 + 销售计划(1.2.3.4) + 运营动作(1.2.3.4) + 战略方向 | |

**关键差异（vs 季报）**：
- 增加H1 vs H2 或 年度 vs 上年度 同比对比
- 增加分店铺半年度/年度收入利润占比分析
- 品类分析按半年度汇总，识别品类战略调整方向
- 千川分析覆盖完整投放周期，评估整体投放效率趋势
- 运营Plan需包含战略方向性调整（产品线增减、渠道策略变更等）
- 增加退货率等年度关键指标的跨期对比

## 工作流步骤

### Step 1: 读取文档结构与内容
```bash
# 获取文档目录
lark-cli docs +fetch --api-version v2 --doc "<doc_url>" --scope outline --max-depth -1

# 获取完整文档（带 block ID，用于后续更新）
lark-cli docs +fetch --api-version v2 --doc "<doc_url>" --detail with-ids
```

### Step 2: 识别数据源
从文档 XML 中识别以下类型的附件引用：
- `<cite file-type="sheets" doc-id="xxx" sheet-id="yyy" title="xxx">` -> 飞书电子表格
- `<cite file-type="file" doc-id="xxx" title="xxx.xlsx">` -> Excel 文件附件

**数据源映射表**：
| 附件类型 | 用途 | 关键sheet/字段 |
|---|---|---|
| 店铺销售数据汇总-财务口径 | 收入/利润/费用/达成率 | 月度汇总sheet |
| 销售及流量汇总-店铺口径 | 渠道结构/流量效率 | 各渠道月度数据 |
| 千川消耗汇总 | 投放消耗/ROI/品类分布 | 商品日明细sheet |
| 抖音视频号明细(Excel) | 分店铺收入利润 | 抖音sheet + 视频号sheet 的"总计"行 |

### Step 3: 提取飞书电子表格数据
```bash
# 先查看工作簿有哪些 sheet
lark-cli sheets +workbook-info --spreadsheet-token "<token>" 

# 读取数据（CSV 格式，带行号标注）
lark-cli sheets +csv-get --spreadsheet-token "<token>" --sheet-id "<sheet_id>" --range "A1:Z100" --json
```
注意：`annotated_csv` 中每行以 `[row=N]` 开头，N 是实际表格行号，不要自己数行。

### Step 4: 下载并解析 Excel 文件附件
```bash
# 下载（必须 cd 到目标目录后用相对路径）
cd /tmp && lark-cli drive +download --file-token "<cite doc-id>" --output ./filename.xlsx --json

# 用 openpyxl 解析
python3 << 'PYEOF'
import openpyxl
wb = openpyxl.load_workbook("/tmp/filename.xlsx", read_only=True, data_only=True)
# 查看所有 sheet
for s in wb.sheetnames:
    print(s)
# 读取特定 sheet（如"抖音"、"视频号"）
ws = wb["抖音"]
for row in ws.iter_rows(min_row=1, max_row=30, values_only=True):
    print(row)
wb.close()
PYEOF
```

**Excel附件关键数据位置**：
- "抖音" sheet：最后一行"总计"行的第4列(销售收入)、第6列(利润总额)
- "视频号" sheet：同上结构
- Q1+Q2合并计算H1时，将两个Excel的总计行数据相加

### Step 5: 数据聚合与分析
用 Python 脚本对原始数据进行多维度聚合：

```python
# 按月聚合（月报核心）
monthly = defaultdict(lambda: {"consume": 0, "amount": 0, "roi": 0})

# 按品类聚合（季报/H1核心）
category = defaultdict(lambda: {"consume": 0, "amount": 0})
# 计算 ROI = amount / consume

# 按季度对比（季报核心）
q1_months = ["2026-01", "2026-02", "2026-03"]
q2_months = ["2026-04", "2026-05", "2026-06"]
# 环比变化 = (Q2 - Q1) / Q1 × 100%

# 按半年度对比（H1报告核心）
h1_consume = q1_consume + q2_consume
h1_amount = q1_amount + q2_amount
h1_roi = h1_amount / h1_consume

# 分店铺聚合（H1报告核心）
douyin_h1 = q1_douyin + q2_douyin
shipinhao_h1 = q1_shipinhao + q2_shipinhao
douyin_pct = douyin_h1 / (douyin_h1 + shipinhao_h1) × 100%
```

**分析维度清单**：
- [ ] 月度趋势：收入/利润/费率/ROI 按月排列
- [ ] 品类分布：各品类消耗占比 + ROI 排名
- [ ] 渠道结构：商品卡/达人/短视频/其他 占比及环比变化
- [ ] 分店铺：抖音 vs 视频号 收入/利润/占比
- [ ] 高ROI标杆 & 低ROI预警商品
- [ ] 流量效率：曝光量/点击率/转化率 同比环比

### Step 6: 逐块更新文档
```bash
# 替换表格单元格中的段落（用 block_replace）
lark-cli docs +update --api-version v2 --doc "<doc_url>" \
  --command block_replace \
  --block-id "<目标段落block_id>" \
  --content '<p>新内容...</p>'
```

**关键规则**：
- 每个单元格中可能有多个段落 block（如运营Review下有多个子项），需要逐个 block_replace
- 替换后旧 block ID 不保证可用，继续操作前需重新 fetch
- 保留原有的 cite 标签（附件引用），不删除不修改
- 如需删除多余的占位 block，用 `block_delete --block-id "id1,id2"` 批量删除

### Step 7: 新增内容到已有段落
当需要在已有单元格中追加新段落（如年度销售进展中增加分店铺数据）时：
```bash
# 方案A：直接在 block_replace 中包含多个 <p> 段落
--content '<p>第一段...</p><p>第二段（新增的分店铺数据）...</p>'

# 方案B：在某个 block 后插入新内容
lark-cli docs +update --api-version v2 --doc "<doc_url>" \
  --command block_insert_after \
  --block-id "<锚点block_id>" \
  --content '<p>新增段落内容</p>'
```

### Step 8: 验证最终结果
```bash
# 重新读取更新后的文档，检查内容是否正确
lark-cli docs +fetch --api-version v2 --doc "<doc_url>" \
  --scope section --start-block-id "<表格标题block_id>" --detail with-ids
```

## 内容撰写指南

### 运营Review 各子板块撰写要点

**【销售收入与运营费用】**：
- 开头一句话总结周期内收入/利润核心数据
- 月度/季度趋势：列出各月收入，标注峰值与低谷
- 品类销售情况：按消耗占比排序，标注ROI，区分高效/低效品类
- 运营费用分析：运费/平台/推广/达人 四项费用，计算费率，标注异常值
- 结尾附 cite 标签引用数据源附件

**【店铺运营】**：
- 渠道结构：商品卡/达人/短视频/其他 各渠道占比 + 环比变化
- 达人渠道分析：达人短视频/直播/图文 细分变化，识别拖累项
- 自营渠道崛起：其他渠道（直播自营+售后）变化趋势
- 流量效率：曝光量/点击率/转化率 同比环比，"降量提质"策略验证

**【千川推广】**：
- 整体数据：总消耗/总成交/整体ROI + 月度趋势
- 品类消耗分布：各品类消耗金额/占比/ROI 排名
- 高ROI标杆商品：列出ROI≥10的商品，标注消耗和成交
- 低ROI/需关注商品：列出ROI<6的商品，标注问题

**【其他】**：
- SOP沉淀：千川投放SOP、达人合作SOP、退货率管控SOP、流量效率SOP

### 运营小结 撰写要点

**好的方面**（3-6点，按报告周期调整数量）：
- 收入/利润增长（同比/环比/达成率）
- 基本盘稳固（商品卡占比、自营渠道增长）
- 流量效率提升（点击率/转化率/单位曝光产出）
- 千川投放效率（整体ROI、高ROI标杆品效应）

**存在的问题**（3-6点）：
- 渠道风险（达人下滑、退货率升高）
- 费用异常（达人费用超支、费率飙升）
- 投放低效（低ROI商品消耗高）
- 月度波动（收入不稳定、ROI下降趋势）

### 运营Plan 撰写要点

**销售/费用预估**：
- 收入预估按月列示，说明预估逻辑（大促/淡季/趋势外推）
- 费用预估分项列示（达人/推广/运费/平台），计算费率目标

**销售计划（1.2.3.4）**：
1. 产品遴选：基于历史ROI数据选择主推品，说明选择理由
2. 权益机制：促销策略（满减/赠品/会员优惠），标注"但会降低利润"
3. 达人费用管控：预算上限 + ROI门槛 + 淘汰机制
4. 推广费用优化：千川聚焦高ROI品类，暂停低ROI品，ROI底线

**运营动作（1.2.3.4）**：
1. 短视频自营内容：目标占比 + 日产数量 + 内容方向
2. 达人合作结构调整：减少短视频带货 + 增加直播 + 组合模式
3. 退货率管控：目标值 + 排查方向 + 优化措施
4. 千川精准投放：预算分配 + 出价策略 + ROI底线

## 注意事项

### XML 转义
- `&` -> `&amp;`，`<` -> `&lt;`，`>` -> `&gt;`
- 标签本身禁止转义，仅转义标签内部的文本内容

### block 操作
- block_replace 只能对同一 block 操作一次，多次修改需合并为一次调用
- block_replace 后旧 block ID 不保证可用，后续操作前需重新 fetch
- block_delete 支持逗号分隔批量删除：`--block-id "id1,id2,id3"`

### cite 标签保留
- 段落内容中的 cite 标签必须保留原样（doc-id、file-type、title 等属性不变）
- 新增段落中引用附件时，cite 标签的 doc-id 必须与文档中已有的附件 token 一致

### 数据准确性
- 所有百分比、ROI、占比等计算结果必须可追溯到附件原始数据
- 分店铺数据从 Excel 附件的"抖音"/"视频号" sheet 的"总计"行提取
- 跨季度/半年度合并时，将各季度的总计行数据相加
